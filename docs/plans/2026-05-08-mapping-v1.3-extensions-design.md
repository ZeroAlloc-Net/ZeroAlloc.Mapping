# ZeroAlloc.Mapping v1.3 — design

**Date:** 2026-05-08
**Scope:** Two backlog items promoted from `docs/backlog.md` for the v1.3 release. Both opt-in/additive — no v1.0/v1.1/v1.2 surface change.

## Items

| ID | Title | Surface |
|---|---|---|
| B8 | Auto-collection overloads | Generator emits `Map(List<TSrc>) → List<TDst>` etc. for every `[Map<,>]` / `[TryMap<,>]` declaration |
| B15 | Duplicate `[MappingCulture]` diagnostic | New ZAMP016 (Warning) when a class declares the marker more than once |

## Goals

- **Each feature opt-in or transparently additive.** v1.0/1.1/1.2 mapping behaviour unchanged for users who don't touch the new surface; B8 adds methods, B15 adds a diagnostic.
- **Zero new allocations on the success path beyond the destination collection itself.** Per-element allocation comes from the leaf `Map`/`TryMap` body, which v1 already certified zero-alloc.
- **Reuse existing machinery.** No new emitters or model layers — extend `MapEmitter`, `TryMapEmitter`, `MapperDiscovery`, `MappingGenerator`.

## Decisions

### 1. B8 — Auto-collection overloads

For every `[Map<TSrc, TDst>]` and `[TryMap<TSrc, TDst>]` declaration on a class, the generator emits four additional overloads with the same name (`Map` or `TryMap`):

#### `[Map<,>]` — emits 4 overloads

```csharp
public static global::System.Collections.Generic.List<TDst> Map(global::System.Collections.Generic.List<TSrc> src)
{
    global::System.ArgumentNullException.ThrowIfNull(src);
    var __dst = new global::System.Collections.Generic.List<TDst>(src.Count);
    for (int i = 0; i < src.Count; i++)
        __dst.Add(Map(src[i]));
    return __dst;
}

public static global::TDst[] Map(global::TSrc[] src)
{
    global::System.ArgumentNullException.ThrowIfNull(src);
    var __dst = new global::TDst[src.Length];
    for (int i = 0; i < src.Length; i++)
        __dst[i] = Map(src[i]);
    return __dst;
}

public static global::System.Collections.Generic.IEnumerable<TDst> Map(global::System.Collections.Generic.IEnumerable<TSrc> src)
{
    global::System.ArgumentNullException.ThrowIfNull(src);
    return global::System.Linq.Enumerable.Select(src, static x => Map(x));
}

public static global::System.Collections.Generic.IReadOnlyList<TDst> Map(global::System.Collections.Generic.IReadOnlyList<TSrc> src)
{
    global::System.ArgumentNullException.ThrowIfNull(src);
    var __dst = new global::TDst[src.Count];
    for (int i = 0; i < src.Count; i++)
        __dst[i] = Map(src[i]);
    return __dst;
}
```

#### `[TryMap<,>]` — emits 4 fallible overloads

Each surfaces collection-element failures via the existing `MappingError.Children` tree pattern (already used by v1's collection-property mapping):

```csharp
public static global::ZeroAlloc.Results.Result<global::System.Collections.Generic.List<TDst>, global::ZeroAlloc.Mapping.MappingError>
    TryMap(global::System.Collections.Generic.List<TSrc> src)
{
    if (src is null) return Result.Failure(new MappingError("mapping.source.null", "(root)"));
    var __dst = new global::System.Collections.Generic.List<TDst>(src.Count);
    var __failures = new global::System.Collections.Generic.List<MappingError>();
    for (int i = 0; i < src.Count; i++)
    {
        var __r = TryMap(src[i]);
        if (__r.IsSuccess) __dst.Add(__r.Value);
        else __failures.Add(new MappingError(
            __r.Error.Code,
            "[" + i + "]" + (__r.Error.PropertyPath == "(root)" ? "" : "." + __r.Error.PropertyPath),
            __r.Error.Reason,
            __r.Error.Children));
    }
    if (__failures.Count > 0)
        return Result.Failure(new MappingError(
            "mapping.collection.elements_failed",
            "(root)",
            __failures.Count + " of " + src.Count + " elements failed",
            __failures));
    return Result.Success(__dst);
}
```

Same shape for `IEnumerable<>`, `IReadOnlyList<>`, and `TSrc[]`.

#### Opt-out

`[SkipCollectionOverloads]` class-level marker:

```csharp
[Map<OrderRequest, Order>]
[SkipCollectionOverloads]
public static partial class M { }
```

When present, the generator emits ONLY the single-element form. Useful when a consumer wants to expose just `Map(OrderRequest)` and not `Map(List<OrderRequest>)` etc.

#### Interaction with other v1.x features

- **`[ReverseMap<,>]`:** already desugars into two `[Map<,>]` decls. Each gets its 4 overloads → 8 total per reverse-map.
- **`[PolymorphicMap<,>]`:** the polymorphic dispatcher is itself a `Map(TBase)` method. Auto-collection emits `Map(List<TBase>) → List<TBaseDto>` etc. that call the dispatcher per element. A `List<Animal>` of mixed `Cat`/`Dog` instances correctly dispatches per element.
- **Update-in-place (`static partial void Map(TSrc, TDst)`):** does NOT get collection overloads — list-pair update semantics are ambiguous (different lengths, element-by-index update). Skipped; revisit if asked.
- **Hooks (`[BeforeMap]`/`[AfterMap]`):** fire only on the leaf `Map(TSrc)` calls, not at the collection level. The collection overload is just a loop wrapper.

#### Allocation

- `List<TSrc> → List<TDst>`: 1 destination list (capacity-presized, zero re-grow). Per element: leaf-Map allocation (already certified).
- `TSrc[] → TDst[]`: 1 destination array. Per element: leaf-Map.
- `IEnumerable<TSrc> → IEnumerable<TDst>`: lazy, zero allocation at call. Each `MoveNext` allocates the LINQ Select state machine via `Enumerable.Select` — measured ~32 B per fresh enumeration. Document.
- `IReadOnlyList<TSrc> → IReadOnlyList<TDst>`: 1 destination array (returned as IReadOnlyList).

Allocation budgets in tests use 256 B / 1000 iter for materialising overloads (covers 50-element lists comfortably) and 64 B for the IEnumerable lazy form.

### 2. B15 — ZAMP016 duplicate `[MappingCulture]`

`[MappingCulture]` is `AllowMultiple = false`, but a class declared in two partial-class parts can carry the attribute on each. The C# compiler does NOT reject this (different syntax sites). v1.2's `MapperDiscovery` silently picks the first via `break` and ignores duplicates.

**Fix:** count the `[MappingCulture]` attribute occurrences. If `> 1` → emit ZAMP016 (Warning). The first-found culture still wins (deterministic by declaration order).

```csharp
public static readonly DiagnosticDescriptor ZAMP016_DuplicateMappingCulture = new(
    id: "ZAMP016",
    title: "Duplicate [MappingCulture] declarations",
    messageFormat: "Class '{0}' declares [MappingCulture] more than once across partial parts; only the first ('{1}') is honoured",
    category: "ZeroAlloc.Mapping",
    defaultSeverity: DiagnosticSeverity.Warning,
    isEnabledByDefault: true);
```

**Detection:** in `MappingGenerator.ReportPerClassDiagnostics`, walk the class's symbol's `GetAttributes()` and count `[MappingCulture]` matches. If `> 1`, fire once per class with the chosen culture name as `{1}` and the class FQN as `{0}`.

This requires `MapperClass.TypeSymbol` to be available for the diagnostic phase. Add `INamedTypeSymbol? TypeSymbol = null` as a trailing optional parameter on `MapperClass`. (Same pattern used for symbol carryover in v1.2's `PolymorphicDecl`.)

|---|---|---|
| **ZAMP016** | Warning | A class declares `[MappingCulture]` more than once (across partial parts); only the first is honoured |

## Out of scope (deferred to v1.4+)

- B3 IQueryable projections (covered by `ZeroAlloc.Specification`).
- B6 cycle detection (defeats zero-alloc).
- B7 private members (out-of-scope-by-design).
- B11 deep cloning (niche).
- True open-generic mappings — `[Map<List<T>, List<U>>]` parametrized by free type parameters. Skipped because C# generic attributes can't cleanly express open-generic instantiation. v1.3's auto-collection-overloads is the pragmatic substitute.
- Update-in-place collection overloads (`Map(List<TSrc>, List<TDst>)`) — list-pair update semantics ambiguous.
- Per-property `[MapPropertyFormat]` (deferred from v1.2).

## Validation strategy

1. **Snapshot tests** for each new emission shape (List/array/IEnumerable/IReadOnlyList × Map/TryMap, plus opt-out and polymorphic interaction).
2. **Diagnostic test** for ZAMP016 + clean-source-doesn't-fire negative-control.
3. **Allocation budgets** for the materialising overloads — extend `AllocationBudgetTests.cs`.
4. **AOT smoke** — extend the AOT binary to exercise one collection overload of each shape.
