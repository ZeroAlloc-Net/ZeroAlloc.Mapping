**Step 4.1: Dotted-path source resolution**

Source: `tests/.../FlatteningTests.cs`. Sections (~200 lines):

1. The pattern: `[MapProperty("Customer.Address.City", "City")]` on a partial `Map` declaration.
2. **Null-handling rule:** target nullability decides operator.

3. **Diagnostic:** ZAMP005 fires at the first missing segment with the cursor type as context.
4. **Edge cases:** dotted path resolving to a private property (filtered out), dotted path on a non-NamedType segment (e.g. tuple — walk halts).

**Step 7.1: `[PolymorphicMap]`/`[PolymorphicTryMap]` switch dispatcher**

Source: `tests/.../PolymorphicMapTests.cs`. Sections (~250 lines):

1. **The pattern** — abstract base + sealed derived, per-derived `[Map<,>]` + `[PolymorphicMap<TBase, TBaseDto>]`.
2. **Generated dispatcher** — switch on runtime type → call leaf `Map`. Show emission.

5. **Degenerate-pair guard:** when `[Map<X, Y>]` + `[PolymorphicMap<X, Y>]` for the same pair, polymorphic emission is suppressed (ZAMP014 still fires).
6. **Performance note:** type-test cost of ~16 B/call beyond the leaf.

| ZAMP010 | Error | Unconsumed source under `[StrictSourceMapping]` |

| ZAMP012 | Error | Update-in-place dest not settable |
| ZAMP013 | Error | `[PolymorphicMap]` with no derived cases |
| ZAMP014 | Warning | `[PolymorphicMap]` over sealed base |
| ZAMP015 | Error | Polymorphic mixed Map/TryMap kinds |
