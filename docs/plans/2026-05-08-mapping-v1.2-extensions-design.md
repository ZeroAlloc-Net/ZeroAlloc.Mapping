# ZeroAlloc.Mapping v1.2 — design

**Date:** 2026-05-08
**Scope:** Three backlog items promoted from `docs/backlog.md` for the v1.2 release. Each is opt-in, additive — no v1.0 / v1.1 surface change.

## Items

| ID | Title | Surface |
|---|---|---|
| B5 | Update-in-place mapping | User-declared partial `static void Map(TSrc, TDst)` overload |
| B9 | FormatProvider customisation | Class-level `[MappingCulture("xx-XX")]` marker |
| B2 | Polymorphic / derived-type mapping | New class-level `[PolymorphicMap<,>]` / `[PolymorphicTryMap<,>]` attributes |

## Goals

- **Each feature opt-in.** v1.0/v1.1 mapping behaviour unchanged for users who don't touch the new attributes or partial overloads.
- **Zero new runtime allocations on the success path.** All extensions resolve at generation time. Polymorphic dispatch adds one `is`/`switch` per call — the dispatcher itself stays alloc-free.
- **Reuse v1's machinery.** No new emitters or model layers — extend `MapEmitter`, `TryMapEmitter`, `MapperDiscovery`, `ConversionResolver`, `Diagnostics`.
- **Each feature ships behind its own snapshot tests.** Diagnostics get positive triggers + clean-source negative-controls.

## Decisions

### 1. B5 — Update-in-place

`[Map<TSrc, TDst>]` continues to emit `static TDst Map(TSrc src)` as before. To get the void-overload form, the user explicitly declares a partial:

```csharp
[Map<OrderRequest, Order>]
public static partial class M
{
    public static partial void Map(OrderRequest src, Order existingDst);
}
```

The generator detects the void overload by signature (`(TSrc, TDst) → void`) on the same class and emits assign-each-property body:

```csharp
public static partial void Map(OrderRequest src, Order existingDst)
{
    global::System.ArgumentNullException.ThrowIfNull(src);
    global::System.ArgumentNullException.ThrowIfNull(existingDst);
    existingDst.Id = src.Id;
    existingDst.Notes = src.Notes;
}
```

`[BeforeMap]` and `[AfterMap]` hooks fire normally — `BeforeMap(src)` before assignment, `AfterMap(src, existingDst)` after.

**Diagnostic surface:**
- **ZAMP012** (Error) — destination type cannot be updated in place because all matched properties are init-only or read-only. Tells the user "use `[Map]` constructor form, or expose settable properties on the destination, or use `[MapperIgnoreTarget]` to drop the offending property."

**TryMap variant:** users declare `static partial Result<Unit, MappingError> TryMap(TSrc, TDst)` and the generator emits a try/catch wrapper — but the trial showed `Unit` adds friction. Reduce v1.2 scope: **`[TryMap]` does NOT support the void-overload update-in-place pattern.** Document explicitly. If the user wants fallible update-in-place, they construct via `TryMap(src)` and copy fields themselves, or write a custom partial that calls `Map(src, existingDst)` inside their own try/catch.

**`[MapperIgnoreTarget]` interaction:** properties marked `[MapperIgnoreTarget]` are skipped in the void-overload (same as in the constructor form). `[MapValue]` constants are assigned (`existingDst.X = "constant"`). `[MapProperty]` renames apply.

**Allocation:** zero. Just per-property assignment. `BudgetMappings.Map(src, dst)` budget: 80 B/call.

### 2. B9 — `[MappingCulture]`

Class-level marker:

```csharp
[Map<Src, Dst>]
[MappingCulture("nl-NL")]
public static partial class DutchMappings { }
```

When the marker is present, `ConversionResolver.Apply` substitutes the named culture for the existing hardcoded `CultureInfo.InvariantCulture`. Affects every `Parse(string, IFormatProvider)` and `ToString(IFormatProvider)` call site emitted by the conversion resolver:

```csharp
// Without [MappingCulture]:
int.Parse(src.Quantity, global::System.Globalization.CultureInfo.InvariantCulture)

// With [MappingCulture("nl-NL")]:
int.Parse(src.Quantity, global::System.Globalization.CultureInfo.GetCultureInfo("nl-NL"))
```

The culture string is passed verbatim to `CultureInfo.GetCultureInfo` at runtime. **No compile-time validation** of the culture name — invalid names throw `CultureNotFoundException` on first call. Documented v1.2 limitation; if real users hit this, v1.3 can add ZAMP015 for unknown cultures.

**Edge case:** `CultureInfo.GetCultureInfo` is itself cached internally by .NET — no per-call allocation.

**Per-property override** (`[MapPropertyFormat]`) is **deferred to v1.3** — class-level covers the common case (one DTO format domain per mapper class).

**Allocation:** unchanged from v1 — `GetCultureInfo` is cached, `Parse` is a single call.

### 3. B2 — Polymorphic mapping

New generic class-level attributes:

```csharp
[AttributeUsage(AttributeTargets.Class, AllowMultiple = true, Inherited = false)]
public sealed class PolymorphicMapAttribute<TBase, TBaseDto> : System.Attribute { }

[AttributeUsage(AttributeTargets.Class, AllowMultiple = true, Inherited = false)]
public sealed class PolymorphicTryMapAttribute<TBase, TBaseDto> : System.Attribute { }
```

Usage:

```csharp
public abstract record Animal(string Name);
public sealed record Cat(string Name, int Lives) : Animal(Name);
public sealed record Dog(string Name, string Breed) : Animal(Name);

public abstract record AnimalDto(string Name);
public sealed record CatDto(string Name, int Lives) : AnimalDto(Name);
public sealed record DogDto(string Name, string Breed) : AnimalDto(Name);

[Map<Cat, CatDto>]
[Map<Dog, DogDto>]
[PolymorphicMap<Animal, AnimalDto>]
public static partial class AnimalMappings { }
```

Generated body:

```csharp
public static global::AnimalDto Map(global::Animal src)
{
    global::System.ArgumentNullException.ThrowIfNull(src);
    return src switch
    {
        global::Cat __c => Map(__c),
        global::Dog __d => Map(__d),
        _ => throw new global::System.InvalidOperationException(
            "No polymorphic mapping for runtime type " + src.GetType().FullName)
    };
}
```

`[PolymorphicTryMap<,>]` variant returns `Result<TBaseDto, MappingError>`; unmatched type returns `Result.Failure(new MappingError("mapping.polymorphic.unhandled_type", "(root)", reason: $"runtime type {src.GetType()} has no declared [Map]"))`.

**Discovery rule:** generator scans the class for `[Map<TDerived, TDerivedDto>]` (or `[TryMap<,>]` for the TryMap variant) where `TDerived` is assignable to `TBase` and `TDerivedDto` is assignable to `TBaseDto`. All matching declarations contribute a switch case.

**Diagnostics:**
- **ZAMP013** (Error) — `[PolymorphicMap<Base, BaseDto>]` declared but no derived `[Map]` declarations exist that produce a `BaseDto`-assignable result. Without cases the dispatcher is dead.
- **ZAMP014** (Warning) — `Base` is `sealed`. Polymorphism over a sealed type is degenerate; the user probably meant `[Map<,>]`.

**Allocation cost:** the dispatcher emits a `switch` expression on `src` — one type-test per case in the worst-bias path. C# compiles `switch (object) { Pattern => … }` to chained `is` checks (or a jump table if the JIT/AOT decides). Zero heap allocation in the dispatcher; the leaf `Map(__cat)` call is the existing zero-alloc body.

**Mixing `[Map]` and `[TryMap]` derived under `[PolymorphicMap]`:** would force the dispatcher to choose how to surface a TryMap failure under a Map signature. Disallow for v1.2 — emit **ZAMP015**: "[PolymorphicMap] cases must all be [Map]; use [PolymorphicTryMap] for fallible dispatch."

## Diagnostics added

| ID | Severity | Trigger |
|---|---|---|
| **ZAMP012** | Error | Update-in-place: destination has no settable properties for any matched param |
| **ZAMP013** | Error | `[PolymorphicMap<Base, BaseDto>]` has no derived `[Map<,>]` cases on the same class |
| **ZAMP014** | Warning | `[PolymorphicMap]` over a `sealed` base type |
| **ZAMP015** | Error | `[PolymorphicMap]` cases include a `[TryMap]` (or vice versa) |

## Out of scope (deferred to v1.3+)

- B3 IQueryable projections — `ZeroAlloc.Specification` covers this.
- B6 cycle detection — defeats zero-alloc.
- B7 private members — out-of-scope-by-design.
- B8 open-generic mappings — generator complexity; revisit if real demand surfaces.
- B11 deep cloning — niche.
- Per-property `[MapPropertyFormat]` — class-level covers the common case for v1.2.
- TryMap-update-in-place — explicit design exclusion (see §1).

## Validation strategy

1. **Snapshot tests** — every new emission shape lands with at least one Verify-driven snapshot.
2. **Diagnostic tests** — ZAMP012/013/014/015 each get a positive trigger plus the existing `Clean_Source_Emits_No_Diagnostics` regression check still passes.
3. **Allocation budgets** — extend `AllocationBudgetTests.cs` with one budget per non-trivial new feature:
   - `Map_UpdateInPlace_WithinBudget` (80 B/call).
   - `Map_With_MappingCulture_WithinBudget` (80 B/call).
   - `Map_Polymorphic_Dispatch_WithinBudget` (96 B/call — slightly looser to absorb the type-test).
4. **AOT smoke** — extend `samples/ZeroAlloc.Mapping.AotSmoke/Program.cs` to exercise one declaration of each feature.
